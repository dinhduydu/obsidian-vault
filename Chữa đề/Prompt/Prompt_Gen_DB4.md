Phân tích toàn bộ các câu sai trong bài làm tiếng Nhật và xuất Markdown thuần cho Obsidian gồm: Summary, Error Analysis, Knowledge Extracted, Learning Profile, Weak Points, Action Next dựa vào template từ file TemplateNoteBookLM (không phân tích lại file này).
Knowledge Extracted chỉ dùng các category: Vocabulary, Kanji, Katakana, Grammar, Particle, Reading, CompoundVerb, Adverb, Conjunction, FixedExpression, Collocation, Keigo, Kenjougo, Demonstratives; mỗi item đúng format ### CATEGORY | ITEM_NAME, chỉ chứa 1 kiến thức.
Mọi item bắt buộc có: Meaning, Trigger, Contrast, Semantic Tag, One-line Summary; Vocabulary thêm Correct Synonym, Distractor1–3, Explanation; Grammar dựa trên Phân Loại Ngữ pháp JLPT và JLPT Grammar Confusion Index để liệt kê các mẫu cùng trường nghĩa, Confusion Set và giải thích vì sao chọn sai; Adverb/Katakana/CompoundVerb/FixedExpression/Collocation/Demonstratives thêm Usage, Example.


Learning Profile phải tạo cho 100% item đã xuất hiện ở Knowledge Extracted, giữ nguyên tên item và format:


CATEGORY | ITEM_NAME


Correct: +N

Wrong: +N

Last Seen: YYYY-MM-DD

Mastery:

Priority:


Ưu tiên phân tích nguyên nhân chọn sai, nhóm lỗi (Grammar Confusion, Trigger, Register, Logic...), chỉ ra dấu hiệu nhận biết (Trigger), bẫy đề và cách phân biệt để tránh lặp lại lỗi dựa vào toàn bộ các câu đã sai.
Không tạo kiến thức tổng quát, không gộp nhiều kiến thức trong một item, luôn dùng đúng tên kiến thức tiếng Nhật cụ thể và tận dụng tối đa dữ liệu từ các file đính kèm.
